Enterprise Fizzbuzz is a demonstration repo for use with learning the ropes with Swimm. It's a humorous take on what should be a 1-line solution to a simple programming problem:

> Print the numbers 1 to 100.
> 
> If the number is a multiple of 3, print "fizz" next to it.
> 
> If the number is a multiple of 5, print "buzz" next to it.
> 
> If the number is a multiple of both 3 and 5, print "fizzbuzz" next to it.

The answer is, of course, to just use the modulus operand to see if 3 or 5 divide evenly into the iteration in the loop, and print accordingly. But we don't stop there, no - we have to make it harder, much much harder than it should be. We're making a core library that handles all the fizzing and buzzing, and then building as many fizzing and buzzing implementations as possible!

That's not enough. We need to come up with at least 20 different callers that call `do_fizz_buzz` internally, so that we can provide fizzing and buzzing in many situations, with lots of complexity, overrides that produce unexpected behavior, and lots of tests.

As we have `TOP_AND_BOTTOM` defined, we understand that we should be wrapping the output in something tabular. During development, we're making this easy to just define on the command line if we need to override (e.g. `TOP_AND_BOTTOM` = "..") or via -D `TOP_AND_BOTTOM` / etc.
